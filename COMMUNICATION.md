# Communication Protocol v1.0

> Owner: 佩丽卡 | 最后更新：2026-06-23

---

## 沟通原则

**Chat 是同步沟通，用于需要即时响应的讨论。**
**Notion 是异步沟通，用于需要持久化的信息和交接。**

> 规则：任何在 Chat 中做出的重要结论，必须在 24 小时内写入 Notion。

---

## 响应时效

| 消息类型 | 预期响应时间 |
|----------|-------------|
| P0 阻断问题 | 30 分钟内 |
| P1 功能问题 | 4 小时内 |
| 普通消息 | 24 小时内 |
| 非紧急问题 | 48 小时内 |

## 交接规范

### 佩丽卡 → 凯尔希

**介质：Notion Task（Owner = 凯尔希）**

```
Task 内容包含：
- Why（为什么做）
- 验收标准（明确、可核查）
- 相关 RFC（链接）
- ETA

完成后：
- 凯尔希附 PR 链接
- 佩丽卡验收
```

### 佩丽卡 → 李织烟

**介质：Notion Task（Owner = 李织烟）**

```
Task 内容包含：
- 知识类型（知识卡片 / 踩坑 / 文档）
- 存放位置（Obsidian 路径）
- 来源参考（PR 链接 / 会议记录 / Chat 记录）

完成后：
- 李织烟附 Obsidian 链接
- 佩丽卡审核
```

### 凯尔希 → 李织烟

**介质：Chat + Notion Task**

```
触发：代码有可复用逻辑
流程：
1. Chat 中 @李织烟 说明有代码值得归档
2. 在 Notion Task 中关联相关代码文件路径
3. 李织烟主动提取归档
```

### 任意 → 佩丽卡

**介质：Chat（简单问题）或 Notion Decision Log（复杂问题）**

```
阻塞升级 → Chat 中说明：
- 阻塞原因
- 已尝试的解决方案
- 需要什么帮助

方向质疑 → 提出 RFC 或在 Task 下评论
```

| 时间 | 内容 | 介质 |

| 每月末 | 月度回顾 + 下月规划 | Notion + Obsidian |

## 三方协作信息流

```
佩丽卡（发起）
    │
    ├── Notion Task ──→ 凯尔希（执行）
    │                        │
    │                        ├── PR ──→ GitHub
    │                        │
    │                        └── Chat ──→ 佩丽卡（完成通知）
    │                                       │
    │                                       ├── 验收
    │                                       │
    │                                       └── Notion Task → 完成
    │                                                    │
    │                                                    ├── Chat ──→ 李织烟（归档通知）
    │                                                    │
    └── Notion Task ──→ 李织烟（归档）                    │
                          │                              │
                          └── Obsidian ──→ 佩丽卡（审核）  │
                                                            │
                                              Obsidian 链接 ──→ Notion Task Proof
```

---

## 信息重复检查

> 每个信息只存在于一个真相源。

| 信息类型 | 唯一存储位置 |
|----------|-------------|
| 任务状态 | Notion Tasks |
| 决策记录 | Notion Decision Log |
| 代码 | GitHub |
| 知识沉淀 | Obsidian |
| 临时讨论 | Chat |
| RFC | Notion RFCs |

**当一个信息出现在多个地方时，以最新更新为准，删除旧位置。**

## 消息格式规范

### Chat 消息格式

**通知类：**
```
[TASK-{id}] {任务名} 完成
链接：{PR/文档链接}
需要佩丽卡验收
```

**升级类：**
```
[BLOKED] {任务名}
原因：{阻塞原因}
已尝试：{尝试的解决方案}
需要：{需要的帮助}
```

**RFC 类：**
```
[RFC-{xxx}] {标题}
状态：{草稿/评审中/通过/拒绝}
链接：{Notion RFC 链接}
请 {凯尔希/李织烟} 填写 {技术评估/知识影响}
```
