## 协作三角

```
佩丽卡（方向）
    │
    │  需求 / 架构 / RFC
    ▼
凯尔希（实现）
    │
    │  代码 / 测试 / 部署
    ▼
李织烟（沉淀）
    │
    │  文档 / 归档 / 知识库
    ▼
佩丽卡（复盘）
```

**单向流动，不逆行。**
凯尔希不对需求提出质疑，只对实现方案提出选择。
李织烟不对方向提出质疑，只对知识整理提出选择。
方向问题永远回传给佩丽卡。

## 决策层级

| 等级 | 类型 | 需要记录 | 决策者 |
|------|------|----------|--------|
| L1 | 日常执行选择 | 否 | 执行者自行判断 |
| L2 | 功能设计变更 | Decision Log | 佩丽卡 |
| L3 | 架构/技术方向 | Decision Log + RFC | 佩丽卡 + 凯尔希 |
| L4 | 团队目标/资源 | Decision Log + Charter修订 | 三方共识 |

**任何 L2 及以上决策，必须在 24 小时内写入 Notion Decision Log。**

---

## 优先级原则

做最重要的事，不是做所有的事。

优先级判断标准（按顺序）：

1. **阻断性问题**（系统崩溃、数据丢失）→ 立即处理
2. **有明确截止日期的工作** → 按截止日期排序
3. **有商业价值的工作** → 佩丽卡判断
4. **长期债务** → Sprint 中留固定比例
